## What SP-0 delivered (a secure multi-tenant shell you can log into)
- **Supabase tenancy spine** (ported/trimmed from the Cinder donor): `workspaces`,
  `workspace_members` (role enum: owner/admin/manager/compliance_officer/accountant/staff),
  `workspace_invites`, `workspace_features`. RLS on every table via `is_workspace_member()`.
- **RPCs:** `create_workspace()` (creator → owner) and `redeem_invite()` (email-bound, SECURITY DEFINER).
- **pgTAP gate:** `rls_isolation_test.sql` (workspaces) + `tenant_isolation_test.sql`
  (members/invites/features) + functional tests for the RPCs. Cross-tenant leakage proven = 0.
- **Express API** (`server/`): config loader, `GET /health`, Supabase client factories incl.
  `userScopedClient` (RLS-respecting), `requireAuth` (JWT → `req.user`), `requireWorkspace`
  (`x-workspace-id` → membership via RLS-scoped client), `requirePermission` (RBAC),
  `GET /api/me`, email/notifications service (safe `{{token}}` interpolation), and a
  module loader + per-workspace feature registry (topo-sort, cycle/missing-dep detection, 404 gating).
- **React SPA** (`web/`): Vite app — Supabase email-OTP `Login`, authed `Shell` calling `/api/me`,
  `App` session gate. Tests use injected deps (no live backend needed).
- **CI** (`.github/workflows/ci.yml`): runs `npm test` + `supabase test db` (the RLS gate).

Full file map is in `AGENTS.md` / the source tree; per-task plan is the SP-0 plan doc.
